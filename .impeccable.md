## Design Context

### Users
Anyone who wants to run AI agents securely on their local machine — from developers and power users managing fleets of bots, to occasional users who just want one agent running without touching a terminal. Context: desktop macOS app, used alongside other dev tools. Users range from technical (Docker-familiar) to non-technical (never seen a terminal).

### Brand Personality
**Secure, minimal setup, deep integration.** ClawPier should feel like a native macOS utility — something that was always supposed to be there. It's the "Docker Desktop for AI agents" but more approachable. The app should feel trustworthy (you're running autonomous agents) without being intimidating.

### Aesthetic Direction
- **Visual tone**: Minimal, refined, macOS-native. Think Arc Browser — clean without being sterile, opinionated without being loud.
- **Theme**: Light mode primary, dark mode supported. Both should feel intentional.
- **Anti-references**: Generic dashboard templates, enterprise SaaS sprawl, CLI-with-a-coat-of-paint.
- **Key reference**: Arc Browser — spatial clarity, confident use of whitespace, typography that breathes, subtle motion that communicates state.

### Design Principles
1. **Disappear into the workflow** — The best tool is one you don't think about. Minimize chrome, maximize content.
2. **Earn trust through clarity** — Users are running autonomous agents. Every state, action, and consequence should be immediately legible.
3. **Progressive depth** — Simple on the surface, powerful underneath. First-time users see what they need; power users discover shortcuts, bulk actions, and advanced config.
4. **Native feel** — Respect macOS conventions. The app should feel like it belongs on the dock next to Finder and Arc.
5. **Light touch** — Prefer subtle over bold. Let content and status speak; decoration should be almost invisible.
